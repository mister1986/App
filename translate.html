Setup: route your PC's sound into a virtual audio input (one-time, not your voice) required once▸

This never records your voice. Browsers can only run live speech recognition on an "input device" — there's no API to hand it system sound directly. The workaround: a virtual audio cable makes your speaker output appear as an input device, so Chrome thinks it's listening to a "mic" when really it's listening to whatever is playing on your PC. You won't speak into anything.

Windows

  1. Install VB-CABLE (free) and reboot if prompted.
  2. Right-click the speaker icon → Sound settings → set Output to "CABLE Input".
  3. Set Input (recording device) to "CABLE Output".
  4. Optional, so you can still hear it yourself: open "CABLE Output" recording properties → Listen tab → check "Listen to this device" → pick your real speakers/headphones.

macOS

  1. Install BlackHole (2ch), free.
  2. In Audio MIDI Setup, create a Multi-Output Device combining BlackHole + your speakers, and set it as system output.
  3. Set your Input device (System Settings → Sound → Input) to BlackHole 2ch — this is the PC-audio feed, not a real mic.

Then here

  1. Pick the language actually being spoken in the audio below.
  2. Click Start Listening. Chrome will still ask for "microphone" permission — that's just the browser's generic label for any input device; approve it to let the page read your PC-audio feed (the virtual cable), not an actual mic.
  3. Click Open Translation Window and drag it to a second monitor or window.

Works in Chrome or Edge on desktop (Web Speech API isn't supported in Firefox/Safari).
