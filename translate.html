Start in 3 steps — no installation or API key recommended▸
  1. Play the video or audio in a Chrome or Edge tab.
  2. Press Start Listening, choose the tab that is playing, and turn on Share tab audio.
  3. Keep the shared tab playing. Spiker transcribes locally with Whisper and displays Persian captions here or in the glass overlay.

Privacy: transcription runs in your browser. No speech-to-text key or Python server is used. The first model download can be large, but the browser normally caches it.

Important: browsers must show the sharing chooser; Spiker cannot capture another tab silently. Upload this file to any normal HTTPS website host. No localhost or application server is required.

Fallback: Chrome speech input / virtual audio cable
Use this only when shared-tab capture is unavailable. Select Chrome speech input / virtual cable under Caption appearance, then route system output through VB-CABLE on Windows or BlackHole on macOS. This fallback may show a microphone-style permission prompt.

Timing: Translation delay is the maximum wait for more context. With adaptive detection enabled, Spiker translates sooner when punctuation, a natural pause, or a complete 1–2 line phrase is detected. Reading time controls the minimum time before the next subtitle replaces it.
Browser Whisper: on Start, Chrome asks the user to choose the YouTube tab and enable Share tab audio. Audio is transcribed locally in the browser with Transformers.js—no Python server or API key. The model downloads once and is cached. WebGPU is fastest; CPU/WASM is the fallback.
Audio note: the embedded YouTube iframe cannot expose audio directly. Press Start, select the YouTube browser tab, and enable Share tab audio. The app never captures a tab without this browser confirmation.
